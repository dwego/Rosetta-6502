# Contributing

Thank you for your interest in contributing to this repository.

Before making any non-trivial change, please first discuss the proposed
change with the project maintainers. This can be done via an issue or
any other appropriate communication channel.

This helps ensure alignment with the project’s direction and avoids
duplicated or conflicting work.

By participating in this project, you agree to follow the Code of Conduct.

---

## Pull Request Process

1. Ensure the project builds cleanly and runs correctly after your changes.

2. Follow the project’s coding, structure, and documentation standards as
   defined in [PROJECT_STYLE.md](PROJECT_STYLE.md).

3. Keep pull requests focused. Each pull request should address a single
   concern or logically related set of changes.

4. Update documentation when behavior, interfaces, or usage are affected.
   Documentation changes are considered part of the contribution.

5. Avoid introducing unnecessary dependencies or build artifacts.

6. All pull requests are subject to review. Feedback is expected and should
   be addressed constructively.

7. Merging is performed by project maintainers.

---

## Code Quality Expectations

- Changes must be readable, maintainable, and consistent with existing code
- Low-level correctness and clarity take precedence over cleverness
- Explicit behavior is preferred over implicit assumptions
- Technical decisions should be justifiable

---

## Communication

Be clear, professional, and respectful in all interactions.

Technical disagreements are expected; personal conflicts are not.
Focus discussions on code, behavior, and design decisions.
